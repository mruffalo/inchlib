On the following charts you can see how inchlib_clust (clustering) and InCHlib (rendering) perform depending on the size of the data.

PC configuration: Kubuntu 13.04 with Chrome 33.0.1750.117 running on Intel Core 2 Duo E6600 2.4GHz, 8GB RAM, classic hard-drive
Clustering time vs. the count of rows

With the increasing count of the data rows the time needed for clustering is growing more or less exponentially. It is not surprising given the nature of clustering and processing all possible row pairs in the data. Be also aware of the PC memory consumption when clustering larger data sets (> 10000).

Clustering time vs. the count of features

The increasing count of the data features doesn't affect the clustering duration as much as the increase of the data rows. The reason is that the complexity of the problem is constant, larger are only the data themselves. Also the consumption of the PC memory is not a problem in that case.

Rendering time vs. the size of the data

Rendering time depends mainly on the count of the data values (e.g., 100 rows * 100 features = 10 000 data values). The rendering time also greatly depends on the PC hardware configuration and used web browser (especially the speed of the JavaScript). We recommend to use the Google Chrome web browser, which is by far the fastest. From the point of the JavaScript speed we would not recommend to use Mozilla Firefox for larger data sets. Be also aware that for longer rendering times (> 30 seconds) the web page can stuck as unresponsive.
